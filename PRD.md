# 🏆 世界杯球迷人格测试 — 产品需求文档

---

## 1. 项目亮点

### 解决什么问题

| 痛点 | 解决方案 |
|------|----------|
| 世界杯期间社交货币匮乏 | 一张专属"球迷身份证"，快速建立看球社交话题 |
| 伪球迷尴尬 | 用测试结果自嘲式认领"懂王型/云球迷"，化解身份焦虑 |
| 缺乏记忆点的互动 | 弹幕式动效 + 定制化卡片，让分享自带传播属性 |

### 好玩在哪里
- **梗文化密集**：题目大量使用球迷黑话（"我奶奶都能进的球"、"越位型人格"）
- **社交货币感强**：每种球迷类型都有精准吐槽金句，支持一键复制文案
- **彩蛋设计**：答完题后不立即显示结果，先来一段"命运揭示"动画再出答案
- **世界杯限定感**：背景动态效果模拟赛场氛围（绿色草坪纹理、黑白五边形足球元素滚动）

## 2. 核心功能与用户流程

### 测试类型

```
[首页] → [选择测试类型]
           ├── 🔮 你是哪种球迷？（10道题）
           └── 🏠 本命球队匹配器（6道题）
```

### 球迷测试流程

```
[首页] → [引导页-规则说明] → [答题页 x10] → [命运揭示动画 3s] → [结果页] → [分享卡片生成] → [二次分享/重新测试]
```

### 球队匹配器流程

```
[首页] → [引导页-规则说明] → [答题页 x6] → [命运揭示动画 3s] → [球队匹配结果] → [分享卡片生成] → [重新测试]
```

### Step 1: 首页
- 全屏 H5 页面，展示所有测试类型入口
- 顶部：标题"足球趣味测试"
- 卡片列表：
  - 🔮 你是哪种球迷？（10道题）
  - 🏠 本命球队匹配器（6道题）
- 点击任意卡片进入对应测试

### Step 2: 引导页（规则说明）
- 3 条简短规则，配合球迷风格插画
- 根据测试类型显示不同规则
- 按钮："开始测试" + "返回首页"

### Step 3: 答题页
- 单题单页，避免用户疲惫
- 顶部：进度条 + 当前题号
- 题目：4 个选项，文字简短有梗
- 每个选项点击后有"选中高亮"动效，然后自动滑入下一题

### Step 4: 命运揭示动画（3 秒）
- 全屏渐黑背景
- 中央出现旋转的足球 + 光效
- 文字："正在分析..."
- 3 秒后自动跳转结果页

### Step 5: 结果页

**球迷测试结果：**
- 超大标题：球迷类型名称 + emoji
- 匹配球队
- 类型描述：精准吐槽
- 能力值展示（雷达图/条形图）
- 按钮："生成专属卡片" / "同步到知乎想法" / "分享给朋友" / "重新测试"

**球队匹配结果：**
- 超大标题：球队名称 + emoji
- 匹配度百分比（0-100%）
- 匹配原因
- 球队描述
- 球队标签（进攻/防守/豪门/青春等）
- 按钮："生成专属卡片" / "重新测试"

### Step 6: 分享卡片
- 自动生成一张横版卡片（适配朋友圈/微博）
- 包含：类型/球队 + 匹配度 + 金句/描述
- 支持保存到相册或直接分享

## 6. 视觉风格与交互设计

### 整体视觉风格
- **主题**：世界杯 + 街头涂鸦 + 霓虹灯牌
- **关键词**：年轻 / 躁动 / 有趣 / 高饱和度

### 色调方案

```
主色调：    #1A1A2E  (深蓝黑 - 世界杯夜景感)
辅助色：    #16213E  (藏青 - 背景层次)
强调色：    #E94560  (荧光红 - CTA按钮/高亮)
辅助强调：  #0F3460  (深蓝 - 卡片背景)
成功色：    #00D9C0  (薄荷绿 - 正确/完成动效)
渐变：      linear-gradient(135deg, #E94560, #FF6B6B)
```

### 动效规范

| 场景 | 动效描述 |
|------|----------|
| 页面入场 | 元素从下往上 fade in，stagger 100ms |
| 选项点击 | 选中项 scale + 边框高亮 |
| 进度条 | 每次答题后 smooth fill |
| 命运揭示 | 足球旋转 + 光效扫射 + 文字打字机效果 |
| 结果页 | 全屏从中心放大 |
| 分享卡片 | 从底部滑入 |

## 7. 技术方案

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| 样式 | Tailwind CSS |
| 动效 | Framer Motion |
| 状态管理 | Zustand |
| 卡片生成 | html2canvas |
| 部署 | Vercel / Netlify / 任意静态托管 |

### 核心模块

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 多测试类型 | Zustand 状态机 | 通过 quizType 区分测试类型 |
| 答题逻辑 | 前端状态机 | 每题独立 state，答题后累加维度分 |
| 结果计算 | 加权算法 | 10题 x 多维度 = 8种球迷类型；6题匹配16支球队 |
| 卡片生成 | html2canvas | 前端直接生成图片 |
